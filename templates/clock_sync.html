Task 4: Berkeley Algorithm

A time synchronization algorithm where clients send their clock values to a master server, which calculates adjustments to synchronize all clocks.

1 Server Time Broadcast: Server broadcasts its current time to all clients
2 Clock Value Calculation: Each client calculates CV (Clock Value) = Local Time - Server Time
3 CV Transmission: Clients send their CV values back to the server
4 Average Calculation: Server calculates the average of all received CVs
5 CAP Calculation: Server computes CAP (Clock Adjustment Parameter) = Average CV - Individual CV
6 Adjustment Broadcasting: Server sends CAP values to respective clients
7 Time Adjustment: Each client adjusts its local time using the received CAP

Task 5: Ricart-Agrawala Mutual Exclusion

A distributed mutual exclusion algorithm using logical timestamps to ensure only one process can access the critical section (marksheet) at a time.

1 Request Critical Section: Process wants to enter critical section, sets state to WANTED
2 Broadcast REQUEST: Send REQUEST(timestamp, process_id) to all other processes
3 Receive REQUEST: Other processes receive request and decide whether to grant permission
4 Permission Logic: Grant OK if: not in CS, not wanting CS, or requesting process has higher priority
5 Collect OKs: Requesting process waits for OK messages from all other processes
6 Enter Critical Section: When all OKs received, process enters critical section (HELD state)
7 Release & Reply: After exiting CS, process replies to all deferred requests
